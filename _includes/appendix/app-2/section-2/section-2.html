Atomes can be used from the command line, including using the following options:

  • General options:

    • "-h" or –help: short help.

    • "-v" or –version: version information.

  • File options:

    • "-awf filename": open Atomes workspace file. \(^*\) \(^{**}\)

    • "-apf filename": open Atomes project file. \(^*\)

    • "-xyz filename": open coordinates in XYZ format. \(^*\)

    • "-pdb filename": open coordinates in PDB format. \(^*\)

    • "-ent filename": open coordinates in PDB format. \(^*\)

    • "-c3d filename": open coordinates in Chem3D format. \(^*\)

    • "-cif filename": open coordinates in CIF format. \(^*\)

    • "-trj filename": open CPMD trajectory. \(^*\)

    • "-xdatcar filename": open VASP trajectory. \(^*\)

    • "-hist filename": open DL_POLY history trajectory. \(^*\)

    • "-ipf filename": open ISAACS project file. \(^*\)

\(^*\) the structure of "filename" is expect to follow the structure introduced with the option (ex: "-xyz" for a file that contains coordinates in the XYZ format, but "filename" is not required to have the corresponding extention (ex: ".xyz").
\(^{**}\) since Atomes has a single workspace only a single workspace can be opened by the command line.
